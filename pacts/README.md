# SoroScan API Pact Contracts

Consumer-driven contract tests keep the SoroScan SDKs and Django API compatible
without requiring full integration test environments.

## Layout

| File pattern | Consumer | Provider |
|---|---|---|
| `soroscan-typescript-sdk-soroscan-api.json` | TypeScript SDK (`sdk/typescript`) | Django API |
| `soroscan-python-sdk-soroscan-api.json` | Python SDK (`sdk/python`) | Django API |

Contracts are generated by consumer tests and verified by producer tests in CI.

## Running locally

### 1. Generate / refresh contracts (consumers)

```bash
# TypeScript SDK
cd sdk/typescript
npm install
npm run test:pact

# Python SDK
cd sdk/python
pip install -e ".[dev]" pact-python
pytest tests/test_pact_consumer.py -v
```

### 2. Verify contracts (producer)

```bash
cd django-backend
pip install -r requirements.txt
DJANGO_SETTINGS_MODULE=soroscan.settings_test \
PACT_PROVIDER_STATES_ENABLED=1 \
pytest soroscan/pact_tests/test_provider_verification.py -v
```

## Provider states

Producer verification uses `POST /_pact/provider-states` (enabled only when
`PACT_PROVIDER_STATES_ENABLED=1`) to seed database fixtures before each
interaction.
